앱은 이미 **빌드 통과 + 24개 페이지 완성** 상태입니다.
아래 2가지만 하면 5분 안에 실제 인터넷에 올라갑니다.

---

## ✅ 1단계: GitHub 저장소 만들기 (2분)

현재 코드가 가리키는 저장소(`nyuwave1-pixel/robot-platform`)가 없어서 "Repository not found"가 뜹니다.

1. https://github.com/new 접속
2. Repository name: `robot-platform` 입력
3. **Private** 선택 (공개해도 무방)
4. "Create repository" 클릭
5. 아무것도 추가하지 말고 (README 체크 해제) 생성만

만든 뒤, 이 폴더에서 아래 명령 한 줄:
```bash
cd "C:\Users\현이\로봇플랫폼개발진행2"
git push -u origin master
```
> 이미 remote가 `nyuwave1-pixel/robot-platform.git`으로 설정돼 있어서 push만 하면 됩니다.

## ✅ 2단계: Vercel 연결 + Firebase 키 (3분)

### A. Vercel 배포
1. https://vercel.com/new 접속 (GitHub 계정으로 로그인)
2. 방금 만든 `robot-platform` 저장소 "Import"
3. Framework: **Next.js** (자동 감지됨)
4. "Deploy" 클릭 → 2분 후 라이브 URL 발급

이 시점에 **홈페이지·게임·디자인은 전부 작동**합니다.
(로그인/DB만 아래 Firebase 키가 있어야 동작)

### B. Firebase 키 (로그인·저장 기능 활성화)
1. https://console.firebase.google.com 접속
2. "프로젝트 추가" → 이름 `robot-platform` (무료 Spark 플랜)
3. 웹 앱 추가(</>) → 설정값 복사
4. Vercel → 프로젝트 → Settings → Environment Variables에 입력:

```
NEXT_PUBLIC_FIREBASE_API_KEY=        (Firebase에서 복사)
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=    (Firebase에서 복사)
NEXT_PUBLIC_FIREBASE_PROJECT_ID=     (Firebase에서 복사)
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET= (Firebase에서 복사)
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID= (Firebase에서 복사)
NEXT_PUBLIC_FIREBASE_APP_ID=         (Firebase에서 복사)
```
5. Vercel에서 "Redeploy" 클릭 → 로그인·DB 활성화 완료

> 키를 안 넣어도 사이트는 **데모 모드**로 정상 작동합니다 (흰 화면 없음).
> 로그인 버튼만 "데모 모드" 안내가 뜹니다.
